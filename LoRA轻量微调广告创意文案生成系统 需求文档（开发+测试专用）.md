# LoRA轻量微调广告创意文案生成系统 需求文档（开发+测试专用）

# 1. 文档概述

## 1.1 文档目的

本文档明确LoRA轻量微调广告创意文案生成系统（以下简称“本系统”）的开发范围、功能需求、技术要求、测试标准及交付物，作为开发人员编码、测试人员执行测试的核心依据，确保系统开发符合预期目标，满足“基于LoRA轻量微调技术，生成行业高转化广告文案”的核心诉求，同时兼顾技术落地可行性与业务实用性。

## 1.2 核心背景

当前广告文案生成存在风格不统一、行业适配性差、生成质量依赖Prompt设计、数据安全难以保障等问题。本系统采用LoRA（Low-Rank Adaptation，低秩适应）轻量微调技术，基于用户提供的<低CTR→高CTR>配对数据集，对大语言模型（LLM）进行小参数微调，使模型具备行业专属高转化广告文案生成能力，同时突出技术含金量，适配固定行业长期文案生产需求及个人简历技术亮点展示需求。

## 1.3 适用范围

本文档适用于本系统的开发人员、测试人员、产品负责人，明确各角色工作边界、任务要求及验收标准；同时为后续系统迭代、维护提供基础参考，不适用于非相关技术及业务场景。

## 核心术语定义

- LoRA轻量微调：低秩适应微调技术，核心是通过调整模型少量“主控旋钮”（低秩矩阵），替代全参数微调，实现轻量化、高效的模型适配，仅修改模型0.4%左右的参数即可达到理想效果，大幅降低训练成本与硬件要求。

- 配对数据集：由用户提供的“低CTR广告文案原文”与“人工优化后的高CTR广告文案”组成的结构化数据集合，是模型微调的核心“专项教材”，用于引导模型学习高转化文案的逻辑与风格。

- LLM：大语言模型，作为本系统的基础模型，通过LoRA微调后，具备行业专属广告文案生成能力。

- CTR：点击通过率，衡量广告文案转化效果的核心指标，低CTR为待优化文案，高CTR为优化后目标文案。

- 领域自适应：通过微调使通用LLM适配特定行业（如电商、教育）的术语、卖点结构及文案风格，从“通才”转变为“行业专家”。

# 2. 核心需求（总目标）

开发一套基于LoRA轻量微调技术的广告创意文案生成系统，实现以下核心目标：

1. 技术层面：基于LLM完成LoRA轻量微调，核心采用“低秩矩阵替换大矩阵修改”的逻辑，仅调整模型关键层参数，实现轻量化训练，支持本地部署，保障数据安全，突出LoRA微调的技术含金量，适配简历技术亮点展示需求。

2. 功能层面：微调后的模型可生成符合目标行业风格、包含行业术语、结构规范的高转化广告文案，生成质量优于纯Prompt方式，且文案风格高度统一、一致性强。

3. 落地层面：适配固定行业（电商、教育、游戏、APP等）的长期、大量文案生产需求，降低人工文案优化成本，同时具备可操作性，平衡训练成本与时间成本。

4. 测试层面：明确微调效果、文案质量、系统性能的测试标准，确保系统稳定运行，满足业务及技术需求。

# 3. 开发需求

## 3.1 功能需求

### 3.1.1 数据集处理模块

- 支持用户上传<低CTR→高CTR>配对数据集，数据集格式支持JSON、JSONL等常见结构化格式，需包含“低CTR原文”“高CTR优化文”两个核心字段，可扩展包含“行业标签”“文案类型”（如标题、详情页文案）等辅助字段。

- 具备数据集清洗功能，可自动剔除重复数据、错别字数据、违规（违反广告法）数据及无关噪音数据，同时支持手动筛选、编辑数据集，确保数据“准、纯、全”，符合微调要求。

- 支持数据集拆分，可按预设比例（如8:2）自动拆分为训练集与验证集，用于模型微调的训练与效果验证，拆分过程可追溯、可调整。

- 数据集格式转换功能，可将用户上传的原始数据，转换为模型微调所需的标准格式（如Chat messages格式），适配LLM与LoRA微调的输入要求。

### 3.1.2 LoRA微调模块

- 基础模型适配：支持主流开源LLM（如Qwen3、Llama等）作为基座模型，优先选择轻量级基座模型（如Qwen3-0.6B），降低本地部署的硬件门槛，同时支持用户指定符合需求的基座模型路径。

- LoRA参数配置：支持用户配置核心微调参数，包括低秩维度r（默认8，可调整为4、16等）、缩放系数α、dropout值（默认0.1，可调整至0.2及以上）、目标模块（优先选择注意力机制的Q、V投影层，可扩展至MLP层）、学习率、训练轮次、批次大小等，参数配置可保存为模板，方便后续复用。

- 微调流程自动化：用户完成数据集上传、参数配置后，系统可自动执行微调流程（数据加载→模型注入LoRA适配器→训练→验证→模型保存），无需手动编写复杂代码，降低操作门槛，同时显示训练进度、损失值变化等实时信息，便于用户监控训练过程。

- 模型保存与导出：微调完成后，支持保存LoRA适配器权重及完整微调模型，可导出为通用格式，支持本地部署（如本地服务器、个人电脑），保障数据不泄露，同时支持模型权重合并，提升推理效率。

- 微调优化：集成LoRA进阶优化能力，支持QLoRA量化（4-bit精度），降低显存占用，使24GB显存的消费级显卡可完成微调；支持分层学习率配置，为不同矩阵设置差异化学习率，提升训练稳定性与收敛速度。

### 3.1.3 文案生成模块

- 基础生成功能：用户输入文案需求（如“电商女装详情页文案”“教育课程推广标题”），微调后的模型可生成符合行业风格、包含行业术语的广告文案，支持生成不同长度、不同侧重点（如卖点突出、情感共鸣）的文案。

- 风格一致性保障：生成的文案需保持高度统一的风格，与用户提供的高CTR配对数据风格一致，避免出现风格混乱、语气不一致的情况，同时准确贴合目标行业的卖点结构。

- 生成质量优化：生成的文案质量需明显优于纯Prompt方式，具体体现为：行业术语使用准确、卖点提炼清晰、转化导向明确，无冗余信息，符合高CTR文案的核心特征。

- 批量生成功能：支持一次性输入多个文案需求，批量生成对应广告文案，适配长期、大量文案生产场景，可导出生成结果（如Excel、TXT格式），提升使用效率。

- 个性化调整：支持用户对生成的文案进行手动编辑、二次优化，同时可将“生成文案→人工优化文案”补充至配对数据集，用于模型二次微调，持续提升生成质量。

### 3.1.4 系统管理模块

- 用户管理：支持单用户登录（适配个人使用场景），可设置登录密码，保障系统使用安全；支持用户信息编辑、密码修改。

- 日志管理：记录系统操作日志（如数据集上传、微调操作、文案生成）、训练日志（如训练参数、训练时间、损失值），日志可查询、导出，便于问题排查与流程追溯。

- 模型管理：支持对已微调的模型进行分类保存、命名、删除、查看详情（如微调参数、适配行业、训练时间），方便用户快速调用不同行业的专用模型。

- 本地部署支持：提供本地部署教程及相关工具，支持用户在本地环境（Windows、Linux系统）部署系统，无需依赖云端服务，保障数据安全，同时提供部署问题排查指引。

## 3.2 技术要求

### 3.2.1 开发语言与框架

- 核心开发语言：Python（3.8及以上版本），适配LLM微调及文案生成的相关库。

- 核心框架：深度学习框架（PyTorch 2.0及以上）、PEFT库（用于LoRA微调）、Transformers库（用于LLM加载与推理）、FastAPI（用于系统接口开发，可选），确保技术栈成熟、可扩展性强。

- 前端框架（可选）：若需简易操作界面，可采用Vue、React等框架，实现可视化操作（数据集上传、参数配置、文案生成），降低操作门槛；若侧重轻量化，可仅开发命令行版本，适配技术人员使用。

### 3.2.2 硬件适配要求

- 训练环境：支持消费级GPU（如NVIDIA RTX 3090/4090，显存≥16GB），通过QLoRA量化可适配4GB显存环境，训练时间控制在几小时内（具体根据数据集大小、参数配置调整），避免过长时间占用硬件资源。

- 部署环境：本地部署支持普通个人电脑（CPU≥i5，内存≥16GB，显存≥8GB）、本地服务器，无需高端硬件配置，确保部署便捷性。

### 3.2.3 数据安全要求

- 数据集安全：用户上传的配对数据集仅保存在本地（或用户指定的本地路径），不进行云端上传、存储，避免数据泄露，支持数据集加密保存（可选）。

- 模型安全：微调后的模型仅保存在用户本地，支持密码保护，防止未授权访问、篡改，确保模型专属使用权。

- 操作安全：日志记录所有敏感操作，支持异常操作提醒，防止误操作导致的数据、模型丢失。

### 3.2.4 性能要求

- 微调性能：数据集规模在100-1000条时，微调时间控制在2-8小时内（根据硬件配置调整），训练过程稳定，无崩溃、卡顿现象，损失值收敛正常（验证集损失值趋于平稳，无明显波动）。

- 生成性能：单条文案生成时间≤3秒，批量生成（≤50条）时间≤100秒，生成过程无卡顿，支持并发生成（≤10条并发），无报错。

- 系统稳定性：连续运行24小时无崩溃，数据集上传、微调、生成等操作成功率≥99%，异常情况（如数据集格式错误、硬件资源不足）有明确提示，便于用户排查。

### 3.2.5 可扩展性要求

- 模型扩展：支持新增、替换基座模型，适配不同LLM的LoRA微调需求，无需大规模修改核心代码。

- 功能扩展：支持新增数据集格式、文案生成模板、微调参数配置项，便于后续根据行业需求迭代优化。

- 行业适配：支持不同固定行业（电商、教育、游戏、APP等）的数据集适配，微调后可快速切换行业专属模型，无需重新开发核心模块。

### 4.2.5 微调效果专项测试

核心测试LoRA微调后模型的效果，对比纯Prompt方式，验证以下指标：

- 行业适配性：生成文案中行业术语使用准确率≥98%，卖点结构符合目标行业规范，无明显不符合行业特点的表述。

- 转化导向：生成文案的核心卖点提炼准确率≥95%，具备高CTR文案的特征（如痛点明确、利益点突出），经人工评估，转化潜力优于纯Prompt生成文案。

- 风格一致性：同一行业、同一风格的文案，一致性≥95%，无语气、风格混乱的情况。

- 泛化能力：使用未参与微调的测试集（同行业），生成文案的质量、风格与训练集适配的文案一致，无过拟合现象（测试集准确率≥85%），原始模型基础能力保留率≥90%。

- 自动评估指标：使用BLEU、ROUGE等指标评估生成文案与高CTR文案的相似度，BLEU值≥0.6，ROUGE-L值≥0.7，优于纯Prompt生成结果。
