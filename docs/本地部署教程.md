# LoRA广告文案生成系统 - 本地部署教程

## 环境要求

| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | i5 4核 | i7 8核+ |
| 内存 | 16GB | 32GB |
| 显存 | 无(CPU模式) | 8GB+ (GPU模式) |
| 系统 | Windows 10 / Ubuntu 20.04+ | Windows 11 / Ubuntu 22.04+ |
| Python | 3.8+ | 3.10+ |

## 安装步骤

### 1. 克隆项目

```bash
git clone <项目地址>
cd ad_helper
```

### 2. 创建虚拟环境（推荐）

```bash
# Windows
python -m venv venv
.\venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate
```

### 3. 安装依赖

```bash
pip install -r requirements.txt
```

**注意**: 如果安装 `bitsandbytes` 失败（Windows兼容性问题），可以跳过：

```bash
pip install torch transformers peft accelerate
pip install fastapi uvicorn sqlalchemy python-multipart
pip install pyyaml tqdm psutil
pip install nltk rouge-score datasets
```

### 4. 创建必要目录

```bash
mkdir -p data/datasets data/splits data/exports logs output models
```

### 5. 下载基座模型（首次运行自动下载）

系统首次启动时会自动从 HuggingFace 下载 `Qwen/Qwen2.5-0.5B` 模型（约1GB）。

如需手动下载：

```bash
from transformers import AutoModelForCausalLM, AutoTokenizer
model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-0.5B")
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2.5-0.5B")
```

## 使用流程

### 1. 上传数据集

支持 JSON 或 JSONL 格式，每条记录包含：

```json
{
  "source_content": "低CTR原文（待优化文案）",
  "target_content": "高CTR优化文（目标文案）",
  "industry_tag": "industry_beauty",
  "copy_type": "title"
}
```

字段说明：
- `source_content`: 低CTR广告文案原文（必填）
- `target_content`: 人工优化后的高CTR文案（必填）
- `industry_tag`: 行业标签（可选）: `industry_beauty` | `industry_fashion` | `industry_game`
- `copy_type`: 文案类型（可选）: `title` | `detail` | `banner` | `short_video`

### 2. 数据预处理

- **清洗**：自动去除重复、违规（包含广告法禁用词）、低质量数据
- **拆分**：按比例拆分为训练集（默认80%）和验证集（默认20%）

### 3. 配置训练参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| LoRA r | 8 | 低秩维度，越大微调能力越强 |
| Alpha | 16 | 缩放系数，通常为 r 的两倍 |
| 学习率 | 2e-4 | 建议范围 1e-4 ~ 3e-4 |
| Epochs | 3 | 训练轮次 |
| Batch Size | 2 | 每设备批次大小 |

### 4. 开始训练

训练过程会显示：
- 当前Step / 总Step
- 训练损失值
- 已用时间 / 预计剩余时间
- GPU/CPU 使用状态（仅GPU模式）

### 5. 生成文案

训练完成后，选择模型并输入低CTR文案进行生成。

## 硬件模式说明

系统会自动检测硬件并选择最优模式：

| 模式 | 触发条件 | 显存要求 | 特点 |
|------|----------|----------|------|
| GPU | 有 NVIDIA 显卡，显存 ≥8GB | ≥8GB | 训练快，推理快 |
| QLoRA | 有 NVIDIA 显卡，显存 4-8GB | ≥4GB | 4bit量化，显存省75% |
| CPU | 无显卡或显存不足 | - | 训练慢，但可运行 |

## 常见问题

### Q1: 启动报错 "ModuleNotFoundError"

**原因**: 依赖未安装完整

**解决**:
```bash
pip install -r requirements.txt
```

### Q2: 模型下载失败

**原因**: 网络问题或未配置代理

**解决**:
```bash
# 设置代理（可选）
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890

# 或手动下载模型后放到 models/ 目录
```

### Q3: Windows 上 bitsandbytes 安装失败

**解决**: 可以跳过此依赖，系统会自动切换到CPU模式

### Q4: 训练时显存不足

**解决**:
1. 减小 Batch Size（改为1）
2. 启用 QLoRA 模式（系统自动选择）
3. 使用更小的基座模型

### Q5: 端口被占用

**解决**: 换一个端口
```bash
python -m uvicorn main:app --port 8001
```

## 技术亮点（简历用）

- **LoRA轻量微调**: 仅调整模型0.4%参数，实现轻量化训练
- **QLoRA量化训练**: 4-bit精度量化，降低显存占用至原来的1/4
- **领域自适应**: 基于行业配对数据微调，模型具备行业专家能力
- **低秩矩阵原理**: 通过低秩矩阵分解替代全参数更新
- **本地部署**: 纯本地运行，无云端依赖，保障数据安全
- **RESTful API**: FastAPI提供标准接口，便于集成扩展
